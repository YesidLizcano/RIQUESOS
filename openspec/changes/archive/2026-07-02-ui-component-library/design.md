# Design: UI Component Library

## Technical Approach

Four-phase rollout: (0) fix the broken Tailwind CSS pipeline so classes actually render, (1) init shadcn/ui and build the shared authenticated layout with sidebar + header, (2) replace every page's raw HTML with shadcn components. Data flow stays RSC-first: pages fetch via Server Actions, pass serializable DTOs as props to Client Components for interactive bits (Data Table sorting, form state).

## Architecture Decisions

| Decision | Choice | Alternatives | Rationale |
|----------|-------|--------------|-----------|
| CSS framework | Tailwind v4 via `npx shadcn@latest init` | Manual Tailwind v3 setup | shadcn init scaffolds globals.css, postcss.config, tailwind.config, and components.json in one step — eliminates the current missing-config gap |
| Component library | shadcn/ui (Radix + Tailwind) | Headless UI, MUI, Chakra | Already proposed; Radix provides accessible primitives, shadcn copies into `src/components/ui/` giving full control; Tailwind-native means no CSS-in-JS bundle |
| Data table | TanStack React Table + shadcn Table primitives | Custom sortable table, AG Grid | TanStack is headless (pairs with shadcn Table), tree-shakeable, supports sorting/pagination; AG Grid is overkill for a backoffice |
| Route layout | Next.js route groups `(dashboard)` + layout.tsx | Per-page layout wrapper, middleware rewrite | Route groups let `/login` render without sidebar while `(dashboard)` pages share one layout — no auth logic duplication |
| Auth guard | RSC-level `getServerSession` + redirect in each page | Middleware-only, AuthProvider context wrapper | Matches existing pattern in all 5 protected pages; middleware can redirect but doesn't pass session data to components — current approach works and is already consistent |
| Icon set | lucide-react | heroicons, react-icons | lucide is shadcn's default icon package; avoids adding another icon dep |

## Data Flow

```
Browser → Next.js Route Group
  ├─ /login (standalone layout, no sidebar)
  │     └─ LoginPage (Client Component)
  │           └─ shadcn Input / Button / Label
  │
  └─ /(dashboard) (shared layout with SidebarProvider)
        ├─ layout.tsx (RSC) → AppSidebar + Breadcrumb + Header
        │
        ├─ page.tsx (RSC) → getMetricas() + getLotes()
        │     └─ MetricCard (RSC, shadcn Card)
        │     └─ LotesTable (Client, TanStack)
        │
        ├─ /lotes (RSC) → getLotes()
        │     └─ LotesTable (Client, TanStack)
        ├─ /ventas (RSC) → getVentas()
        │     └─ VentasTable (Client, TanStack)
        ├─ /clientes (RSC) → getClientes()
        │     └─ ClientesTable (Client, TanStack)
        └─ /gastos (RSC) → getGastos()
              └─ GastosTable (Client, TanStack)
```

## Testing Strategy

| Layer | What to Test | Approach |
|-------|-------------|----------|
| Unit | `cn()` utility | Vitest: verify class merging |
| Unit | DataTable component | Vitest + RTL: renders columns, sorts on header click |
| Unit | MetricCard variants | Vitest + RTL: renders title/value, applies variant class |
| Integration | Dashboard page renders | E2E (Playwright): full page load with mock data |
| Integration | Auth guard redirects | E2E: unauthenticated → `/login` redirect |

## Migration / Rollout

Phased rollout matching proposal phases:
1. **Phase 0**: Run `shadcn init` → verify `npm run build` succeeds and all pages render styled
2. **Phase 1**: Create `(dashboard)` route group + layout → move pages → add sidebar → verify navigation
3. **Phase 2**: Replace raw HTML with shadcn components per page → verify each page independently

Rollback: revert commit per phase. Each phase is a self-contained deployable unit.
